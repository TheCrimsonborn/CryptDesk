WORKFLOW

Remote desktop support in one clean path.

CryptDesk is built for remote support, desktop sharing, and direct remote access. Share a route, verify identity, then control only if the person on the other end wants that to happen.

01

Expose a route

The host starts sharing and sends an IP address and port to the viewer.

02

Connect directly

The viewer dials the target route with an optional shared secret for extra assurance.

03

Verify the safety code

Both sides compare the visible fingerprint before they trust the session.

04

Enable control deliberately

The host allows remote input, and only then can the viewer send mouse or keyboard events.

TRUST MODEL

No hidden handoff. No silent control.

The app keeps its remote access trust cues visible instead of burying them behind setup friction. The session should stay understandable to the person sharing their machine.

End-to-end session keys

X25519 is used for ephemeral key exchange and the resulting remote desktop transport is encrypted with AES-GCM.

Human-verifiable fingerprint

The safety code gives both peers a simple way to confirm they are looking at the same remote desktop session.

Optional shared secret

An extra passphrase can be mixed into session derivation when a remote access route is shared through weaker channels.

Explicit host control gate

Remote input is blocked until the host toggles approval, even if the viewer already sees the screen or desktop preview.
